PLS-SEM 完整操作流程，含反映型與形成型測量模型

1. PLS-SEM 簡介與 CB-SEM 的比較

什麼是 PLS-SEM？

偏最小平方法結構方程模型 (Partial Least Squares SEM) 是一種基於變異數 (variance-based) 的 SEM 方法，旨在最大化依變數的解釋變異量 (R²)。相對於 CB-SEM（如 AMOS、LISREL）以共變異數為基礎，PLS-SEM 更適合預測導向的研究。

比較面向 CB-SEM (AMOS / LISREL) PLS-SEM (SmartPLS)
估計方法 共變異數為基礎 (Covariance-based) 變異數為基礎 (Variance-based)
研究目標 理論驗證 (Theory confirmation) 預測與理論發展 (Prediction & theory development)
樣本需求 通常需要較大樣本 (n ≥ 200) 可處理較小樣本
常態性假設 要求多元常態分配 無分配假設 (Non-parametric)
模型複雜度 適合中等複雜模型 可處理高度複雜模型
測量模型 僅支援反映型 同時支援反映型與形成型
模型適配度 提供完整適配度指標 (χ², CFI, RMSEA) 有限適配指標 (SRMR, NFI); 以 R² 為主

何時選擇 PLS-SEM？

  • 研究目標偏向預測而非僅驗證理論
  • 模型包含形成型建構 (formative constructs)
  • 樣本量較小或資料不符常態分配
  • 模型結構複雜（多建構、多路徑）
  • 探索性研究或理論尚在發展階段

2. 反映型 vs. 形成型測量模型

這是 PLS-SEM 中最關鍵的概念區分。選擇錯誤的測量模型類型會導致嚴重的統計結論偏誤。

反映型指標 (Reflective Indicators)

  • 因果方向：建構 → 指標
  • 指標是建構的「反映/表現」
  • 指標之間高度相關（可互換）
  • 移除一個指標不改變建構意義
  • 例：滿意度 → 「我很滿意」「我會推薦」「我會再購買」

形成型指標 (Formative Indicators)

  • 因果方向：指標 → 建構
  • 指標共同「形成/組成」建構
  • 指標之間不一定相關（各代表不同面向）
  • 移除一個指標會改變建構定義
  • 例：社經地位 ← 教育程度 + 收入 + 職業聲望

關鍵判別準則

判別準則 反映型 (Reflective) 形成型 (Formative)
因果方向 建構 → 指標 (outward) 指標 → 建構 (inward)
指標間相關 高（期望高相關） 不一定（可高可低）
指標可互換性 可互換（刪除不影響建構定義） 不可互換（每個指標捕捉獨特面向）
參數名稱 外部負荷量 (Outer Loadings, λ) 外部權重 (Outer Weights, w)
評估重點 內部一致性、聚合效度、區辨效度 共線性 (VIF)、外部權重顯著性、聚合效度（冗餘分析）
SmartPLS 設定 Mode A（預設） Mode B

判斷決策流程

常見錯誤

許多研究者將所有建構都設為反映型 (Mode A)。如果你的指標是「組合」出建構的不同面向（如行銷組合的 4P），應該使用形成型 (Mode B)。在 SmartPLS 中，需要手動更改指標模式。

3. SmartPLS 操作步驟

建立新專案

  1. 開啟 SmartPLS → File → New Project
  2. 輸入專案名稱（建議使用英文，避免特殊字元）
  3. 選擇儲存路徑
  4. 點擊 OK 建立專案

SmartPLS 4 支援直接建立或從範本匯入。每個專案可包含多個模型。

匯入資料

  1. 準備 CSV 檔案（推薦）或 Excel 檔案
  2. 第一行必須為變數名稱（欄位標題）
  3. 在專案上右鍵 → Import Data → 選擇檔案
  4. 檢查資料預覽，確認變數類型（數值/類別）
  5. 點擊 Import 完成匯入

資料準備注意事項

  • 確保無空白欄位名稱
  • 遺漏值可選擇 Mean Replacement 或 Case-wise Deletion
  • 反向題需事先進行反向計分
  • 建議使用統一的變數命名規則（如 SAT1, SAT2, LOY1）

繪製路徑模型

  1. 雙擊專案中的空白模型 → 開啟建模畫布
  2. 從左側面板拖曳「Latent Variable」到畫布中（代表建構）
  3. 雙擊建構命名（如 Satisfaction, Loyalty）
  4. 從資料集拖曳指標（觀察變數）到對應的建構上
  5. 使用箭頭工具繪製建構之間的路徑關係
  6. 箭頭從自變數指向依變數

設定形成型建構（重要！）

預設所有建構為反映型 (Mode A)。若需要形成型：右鍵點擊建構 → Properties → 將 Indicator Treatment 改為 Mode B。

執行 PLS 演算法與 Bootstrapping

  1. PLS Algorithm（基本估計）： Calculate → PLS Algorithm → 使用預設設定 → Start Calculation
  2. Bootstrapping（檢定顯著性）： Calculate → Bootstrapping → Subsamples ≥ 5000 → Start Calculation
  3. Blindfolding（預測相關性 Q²）： Calculate → Blindfolding → Omission distance = 7（預設）

Bootstrapping 設定建議

  • Subsamples: 5,000 次（Hair et al. 建議最少）
  • Significance Level: 0.05（雙尾）
  • Complete Bootstrapping: 選擇此選項可獲得所有參數之信賴區間

白話解讀模型圖上的數值

  • 建構旁的紅色數字（如 0.82, 0.88）= Outer Loadings：每個問卷題目跟它所屬的建構有多「緊密」。越接近 1 代表這題越能代表這個概念。
  • 箭頭上的紫色數字（如 0.76, 0.67）= Path Coefficients：兩個建構之間的影響力強度。0.76 代表服務品質對滿意度有很強的正向影響。

Bootstrapping 後模型圖的變化

執行 Bootstrapping 後，你可以在模型圖上切換顯示 t-value 或 p-value（點選上方工具列的下拉選單）。如果路徑係數的 p < 0.05（或 t > 1.96），代表該路徑「統計上顯著」，也就是這個影響關係不太可能只是偶然出現的。

4. 反映型測量模型評估與公式

反映型測量模型的評估包含四個面向，必須依序檢視。以下每個指標都附上公式與白話解釋：

指標信度 (Indicator Reliability)

白話解釋

想像你用 4 題問卷來測量「滿意度」。Outer Loading 就是看每一題到底有多少分量是真正在量「滿意度」，又有多少只是測量誤差。Loading = 0.85 代表這題有 72%（0.85²）的變異量來自「滿意度」本身，只有 28% 是雜訊。

Indicator Reliability = λi² 其中 λᵢ 為第 i 個指標的外部負荷量。λᵢ² 代表建構能解釋該指標多少比例的變異。門檻：λᵢ ≥ 0.708 → λᵢ² ≥ 0.50
負荷量範圍判斷行動
≥ 0.708保留符合標準（λ² ≥ 0.50）
0.40 – 0.70考慮刪除若刪除後能提高 CR 或 AVE 則刪除
< 0.40應刪除該指標解釋力嚴重不足

SmartPLS 查看位置：PLS Algorithm → Outer Loadings

內部一致性信度 (Internal Consistency Reliability)

白話解釋

內部一致性回答的問題是：「同一組題目之間是不是都在量同一件事？」如果 4 題都在量滿意度，受試者在這 4 題的回答應該很一致（都高或都低）。α 和 CR 就是在衡量這種「一致性」的程度。

Cronbach's α = (k / (k-1)) × (1 - ∑σi² / σtotal²) k = 題目數；σᵢ² = 每題的變異數；σ²total = 總分的變異數。假設每題貢獻相等（等權重）
Composite Reliability (CR) = (∑λi)² / [(∑λi)² + ∑(1 - λi²)] λᵢ = 第 i 個指標的負荷量。CR 考慮每題不同的負荷量，比 α 更精確。在 PLS-SEM 中優先採用 CR
指標門檻值白話意義
Cronbach's α ≥ 0.70 假設每題一樣重要，計算的一致性下限
CR 0.70 – 0.95 允許每題重要性不同，更精確的一致性指標
rho_A (ρA) ≥ 0.70 α 和 CR 之間的最佳折衷估計

注意：CR 不應超過 0.95！超過代表題目之間太過相似（幾乎在問同一件事），應考慮刪除冗餘題目。

SmartPLS 查看位置：PLS Algorithm → Construct Reliability and Validity

聚合效度 (Convergent Validity)

白話解釋

聚合效度問的是：「這組題目是不是真的都聚集在同一個概念上？」AVE 計算的是：平均而言，建構能「吃掉」每題變異量的百分比。AVE = 0.60 代表建構平均解釋了每題 60% 的回答差異，剩下 40% 是測量誤差。如果 AVE < 0.50，代表誤差比建構本身的解釋力還大——題目可能不夠好。

AVE = (∑ λi²) / n n = 指標數量；λᵢ = 各指標的 Outer Loading。先把每個 Loading 平方，再取平均值。門檻：AVE ≥ 0.50

計算範例

假設 Satisfaction 有 3 題，Loadings 分別為 0.91, 0.87, 0.84

AVE = (0.91² + 0.87² + 0.84²) / 3 = (0.828 + 0.757 + 0.706) / 3 = 0.764 ≥ 0.50 ✓

SmartPLS 查看位置：PLS Algorithm → Construct Reliability and Validity → AVE 欄位（最右邊的欄）

區辨效度 (Discriminant Validity)

白話解釋

區辨效度回答的問題是：「滿意度和忠誠度真的是兩個不同的概念嗎？還是其實在量同一件事？」如果兩個建構的題目回答模式太過相似，我們就無法確定它們是不同的概念。HTMT 就像在算「兩組題目到底有多像」——越像（越接近 1）代表越難區分。

HTMT = (heterotrait-heteromethod correlations) / geometric mean of (monotrait-heteromethod correlations) 白話：不同建構的題目之間的「平均相關」÷ 同一建構題目之間的「平均相關」。如果比值接近 1，代表跨建構的相關跟建構內部一樣高——區辨效度有問題。
Fornell-Larcker: √AVEi > rij (for all j ≠ i) 白話：一個建構的 √AVE 要大於它跟所有其他建構的相關係數。意思是「一個概念跟自己的題目的關係」應該比「跟別人的題目的關係」更強。

PLS-SEM 提供三種檢驗方法：

方法準則SmartPLS 位置
交叉負荷量 (Cross-Loadings) 每個指標在其所屬建構上的負荷量應高於在其他建構上的負荷量 Discriminant Validity → Cross Loadings
Fornell-Larcker 每個建構的 √AVE 應大於該建構與其他建構之間的相關係數 Discriminant Validity → Fornell-Larcker Criterion
HTMT HTMT < 0.90（保守標準 < 0.85） Discriminant Validity → HTMT

HTMT 為首選方法

Henseler et al. (2015) 指出 HTMT (Heterotrait-Monotrait Ratio) 比 Fornell-Larcker 和交叉負荷量更能有效偵測區辨效度問題。Hair et al. (2019) 建議以 HTMT 作為主要判斷標準。

5. 形成型測量模型評估

重要差異

形成型建構不能使用反映型的評估標準（如 Cronbach's α、CR、AVE）！因為形成型指標不一定相關，使用內部一致性指標沒有意義。

聚合效度：冗餘分析 (Redundancy Analysis)

白話解釋

形成型建構不能用 AVE 來檢驗聚合效度（因為指標之間不需要高度相關），所以我們換一個方式：額外問受試者一道「總結性」的題目，直接問他對這個概念的整體感受，然後看形成型建構跟這道總結題的關聯有多強。如果關聯夠高（路徑係數 ≥ 0.70），就表示那些指標組合起來確實在衡量我們想要衡量的東西。

操作步驟

  1. 建立一個與形成型建構測量相同概念的全域性單一指標 (global single item)
  2. 將形成型建構與該全域指標之間建立路徑
  3. 路徑係數 ≥ 0.70（表示兩者的相關 ≥ 0.70）

例如：若建構為「工作壓力」（由5個壓力來源指標形成），需加入一題「整體而言，我感受到的工作壓力程度」作為全域指標進行冗餘分析。

SmartPLS 查看位置：PLS Algorithm → Path Coefficients（查看形成型建構到全域指標的路徑係數）

共線性診斷 (Collinearity Assessment)

白話解釋

共線性就是「兩個指標太過相似，幾乎在量同一件事」。例如「月收入」和「年收入」放在同一個形成型建構裡，它們幾乎完全重疊。VIF 衡量的是一個指標被其他指標「預測」的程度——VIF 越高，代表這個指標跟其他指標越像，會讓統計結果變得不穩定。

VIFi = 1 / (1 - Ri²) Rᵢ² = 用其他所有指標去預測第 i 個指標的 R²。如果其他指標能完美預測它（R²→1），VIF→∞；如果完全不相關（R²=0），VIF=1。門檻：VIF < 5（理想 < 3）
VIF判斷
< 3理想，無共線性問題
3 – 5可接受，但需注意
≥ 5嚴重共線性，需處理

共線性過高時的處理方式

  • 合併高度相關的指標為單一指標
  • 將高共線性指標建立為高階建構 (Higher-Order Construct)
  • 最後手段：刪除導致共線性的指標（但需謹慎，因為會改變建構定義）

SmartPLS 查看位置：PLS Algorithm → Outer VIF Values

外部權重顯著性與相關性

白話解釋

外部權重 (Outer Weight) 是每個指標對形成型建構的「相對貢獻度」。想像你在組一支籃球隊（建構），每位球員（指標）的貢獻不同——有人是主力得分手，有人負責防守。外部權重就是告訴你每位球員對球隊整體戰力的相對重要性。透過 Bootstrapping 檢驗權重是否顯著，就是在確認這位球員是否真的對球隊有貢獻，還是只是板凳球員。如果權重不顯著，再看外部負荷量（絕對貢獻）——如果連絕對貢獻都很低，就要考慮是否該把這位球員移出名單。

6. 結構模型評估

確認測量模型通過評估後，才進行結構模型（路徑關係）的評估。

結構模型評估五步驟

步驟 評估內容 門檻 / 判斷準則 SmartPLS 位置
1 共線性 (VIF) Inner VIF < 5（理想 < 3） PLS Algorithm → Inner VIF Values
2 路徑係數顯著性 p < 0.05, |t| > 1.96 Bootstrapping → Path Coefficients
3 R²（解釋變異量） 0.75 強、0.50 中、0.25 弱 PLS Algorithm → R Square
4 f²（效果量） 0.02 小、0.15 中、0.35 大 PLS Algorithm → f Square
5 Q²（預測相關性） Q² > 0 有預測相關性；0.02/0.15/0.35 Blindfolding → Construct Cross-validated Redundancy

R²（判定係數）白話解釋與公式

R² 回答最直覺的問題：「我的模型到底解釋了多少？」R² = 0.58 代表你的自變數共同解釋了依變數 58% 的變異量。剩下的 42% 是由你沒有放進模型的其他因素所造成的。想像成考試：你的模型答對了 58 分（滿分 100），還有 42 分的差異是其他原因造成的。

f²（效果量）白話解釋與公式

f² 回答的問題是：「如果我把某一個自變數從模型中拿掉，R² 會掉多少？」它衡量的是一個自變數「獨特的貢獻」。f² = 0.35 代表這個自變數對模型有很大的獨特貢獻；f² = 0.02 代表幾乎可有可無。想像成一個團隊：f² 就是看「少了這個人，團隊績效會掉多少」。

f² = (R²included - R²excluded) / (1 - R²included) R²included = 完整模型的 R²；R²excluded = 移除該自變數後的 R²。分母 (1-R²) 是「未解釋的變異」作為基準

計算範例

完整模型 R² = 0.58；移除「服務品質」後 R² = 0.25

f² = (0.58 - 0.25) / (1 - 0.58) = 0.33 / 0.42 = 0.786 → 大效果量 (large)

Q²（預測相關性）白話解釋

Q² 回答的問題是：「我的模型不只能解釋現有的資料，還能預測新的資料嗎？」它用 Blindfolding（遮蔽法）技術：故意遮住一部分資料，用剩下的資料去預測被遮住的部分。如果預測得比「瞎猜」好（Q² > 0），代表模型具有預測能力。Q² 越高，預測越精準。

SmartPLS 操作：Calculate → Blindfolding → Omission distance = 7 → 查看 Construct Crossvalidated Redundancy

9. 隨堂測驗

測試你對 PLS-SEM 的理解

Q1：「社經地位」由教育程度、收入、職業聲望三個指標組成。這個建構應該設定為？

反映型 (Mode A)，因為這三個指標都與社經地位有關
形成型 (Mode B)，因為這三個指標共同「組成」社經地位的定義
兩者皆可，視研究者偏好
正確！教育程度、收入、職業聲望各代表社經地位的不同面向，是「組合」出建構的定義，因此為形成型。移除任一指標會改變建構的概念意義。

Q2：一位研究者發現其形成型建構中兩個指標的 VIF 分別為 5.8 和 6.2。最適當的處理方式為？

直接刪除兩個高 VIF 指標
忽略不管，因為形成型指標不需要檢查共線性
考慮合併這兩個高度相關的指標，或建立高階建構
正確！VIF > 5 表示嚴重共線性。直接刪除形成型指標會改變建構定義，因此優先考慮合併指標或建立高階建構。形成型指標的共線性檢查是必要的評估步驟。

Q3：評估反映型測量模型的區辨效度時，目前最被推薦的方法是？

交叉負荷量 (Cross-Loadings)
Fornell-Larcker 準則
HTMT (Heterotrait-Monotrait Ratio)
正確！Henseler et al. (2015) 的研究指出 HTMT 在偵測區辨效度問題上優於 Fornell-Larcker 和交叉負荷量。Hair et al. (2019) 建議 HTMT < 0.90（保守標準 < 0.85）。

Q4：一個反映型建構的 Composite Reliability (CR) 為 0.97。這個結果表示？

信度非常好，無需擔心
CR 過高 (> 0.95)，指標可能存在冗餘問題
應改用 Cronbach's α 來評估
正確！CR > 0.95 表示指標之間可能過度重複，測量了幾乎相同的內容。建議檢視是否可刪除部分冗餘指標。理想的 CR 範圍為 0.70 – 0.95。

Q5：在結構模型評估中，R² = 0.52 代表什麼意思？

自變數共解釋了依變數 52% 的變異量，屬於中等解釋力
模型適配度為 52%，可以接受
有 52% 的路徑係數是顯著的
正確！R² 代表模型中的自變數（外生建構）能解釋依變數（內生建構）多少比例的變異量。R² = 0.52 屬於中等水準（0.25 弱、0.50 中、0.75 強）。

10. 重要術語

PLS-SEM (偏最小平方法結構方程模型)

一種基於變異數的 SEM 方法，透過迭代演算法估計潛在變數分數，目標在於最大化依變數的 R²。相較於 CB-SEM，對樣本大小和分配假設的要求較低，且能同時處理反映型和形成型建構。

反映型指標 (Reflective Indicators)

建構導致指標出現的測量模型。指標被視為建構的「反映」或「表現」，彼此之間應高度相關。在 SmartPLS 中設定為 Mode A。評估使用 Outer Loadings、CR、AVE、HTMT。

形成型指標 (Formative Indicators)

指標共同形成或定義建構的測量模型。每個指標捕捉建構的獨特面向，彼此不一定相關。在 SmartPLS 中需手動設定為 Mode B。評估使用 VIF、Outer Weights 顯著性、冗餘分析。刪除指標會改變建構的概念意義。

HTMT (異質-單質特質比)

評估反映型建構之間區辨效度的方法。計算不同建構指標間相關的平均值（heterotrait-heteromethod）與同一建構指標間相關的幾何平均值（monotrait-heteromethod）之比率。HTMT < 0.90 表示具有區辨效度（保守標準 < 0.85）。

Bootstrapping（拔靴法）

一種非參數統計推論方法，透過重複抽樣（建議至少 5,000 次）來估計標準誤和信賴區間，進而檢定路徑係數和外部權重的統計顯著性。PLS-SEM 因為不假設常態分配，必須使用 Bootstrapping 進行推論統計。

f² 效果量 (Effect Size)

衡量特定自變數對依變數 R² 的貢獻程度。計算方式為移除該自變數前後 R² 的差異除以（1 - R²included）。判斷標準：0.02 小效果、0.15 中效果、0.35 大效果。f² ≈ 0 表示該自變數對依變數幾乎沒有實質影響。

Q² 預測相關性 (Predictive Relevance)

使用 Blindfolding 程序（交叉驗證冗餘法）計算，衡量模型對內生建構指標的預測能力。Q² > 0 表示模型具有預測相關性。判斷標準與 f² 相同：0.02 小、0.15 中、0.35 大。Omission distance 建議設為 7。
