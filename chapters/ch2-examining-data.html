遺漏值、離群值、常態性與資料轉換

一、遺漏值（Missing Data）

白話解釋：為什麼遺漏值重要？

想像你做了一份100人的問卷調查，但其中30人跳過了「收入」這題。如果你直接忽略這些人，你的分析結果可能只代表「願意回答收入問題的人」，而不是全部受訪者。更糟的是——如果高收入者更傾向跳過這題（怕被知道），你的資料就會系統性地偏離真實情況。

Imagine you conducted a 100-person survey, but 30 people skipped the "income" question. If you simply ignore them, your results may only represent "people willing to answer about income," not all respondents. Worse — if high-income people tend to skip this question (to avoid disclosure), your data systematically deviates from reality.

遺漏值的處理是分析前最重要的步驟之一。遺漏值不只會縮小樣本量，更可能導致偏誤的統計推論。處理遺漏值的第一步是了解其「遺漏的機制」。

Handling missing data is one of the most critical pre-analysis steps. Missing data not only reduces sample size but can also lead to biased statistical inference. The first step is understanding the "mechanism" of missingness.

遺漏值的三種類型

MCAR：完全隨機遺漏

定義：遺漏的發生與任何變數（包含自身）都完全無關。

例子：問卷被風吹走、資料輸入時不小心跳過。

影響：最理想的情況——不會產生偏誤，只損失統計檢力。

MCAR is the ideal case — missingness is completely unrelated to any variable.

Definition: Missingness is completely unrelated to any variable, including itself.

Example: Survey blown away by wind, data entry accidentally skipped.

Impact: Best case — no bias, only loss of statistical power.

MAR：隨機遺漏

定義：遺漏的發生與其他觀察到的變數有關，但與遺漏值本身無關。

例子：女性較傾向不回答收入問題（遺漏與「性別」有關，但與收入本身無關）。

影響：可以透過適當的插補方法處理。

MAR means missingness depends on other observed variables but not the missing value itself.

Definition: Missingness is related to other observed variables but not to the missing value itself.

Example: Women may be less likely to report income (missingness relates to "gender" but not to income itself).

Impact: Can be handled with appropriate imputation methods.

MNAR：非隨機遺漏

定義：遺漏的發生與遺漏值本身有關。

例子：高收入者不回答收入問題（正是因為收入高才不回答）。

影響：最棘手的情況——無論用什麼方法都可能產生偏誤，需要特殊建模處理。

MNAR is the most problematic — missingness depends on the unobserved value itself.

Definition: Missingness is related to the missing value itself.

Example: High-income individuals not reporting income (they skip precisely because income is high).

Impact: Most problematic — any method may produce bias; requires special modeling.

遺漏值處理方法

完整刪除法（Listwise Deletion / Complete Case Analysis）

只保留所有變數都有完整資料的個案，任何一個變數有遺漏就整筆刪除。

  • 優點：操作最簡單、最常用；在 MCAR 條件下不會產生偏誤
  • 缺點：可能大幅損失樣本量；若非 MCAR 則產生偏誤
  • 適用時機：遺漏比例低（< 5%）且符合 MCAR

Listwise Deletion (Complete Case Analysis)

Keep only cases with complete data on all variables; delete any case with any missing value.

  • Pros: Simplest, most common; unbiased under MCAR
  • Cons: Can drastically reduce sample size; biased if not MCAR
  • When to use: Low missing rate (< 5%) and MCAR holds

配對刪除法（Pairwise Deletion / Available Case Analysis）

計算每一對變數的統計量時，使用該對變數上都有完整資料的所有個案。

  • 優點：保留更多資料，每個統計量使用最大可用樣本
  • 缺點：不同統計量的樣本大小不一致；可能產生不正定的相關矩陣
  • 適用時機：以相關或共變異數為基礎的分析

Pairwise Deletion (Available Case Analysis)

For each pair of variables, use all cases with complete data on both variables.

  • Pros: Retains more data; each statistic uses maximum available sample
  • Cons: Different sample sizes for different statistics; may produce non-positive-definite correlation matrices
  • When to use: Correlation or covariance-based analyses

插補法（Imputation Methods）

用估計值來取代遺漏值，保留完整的樣本大小。

  • 平均數插補：用該變數的平均數取代遺漏值。簡單但會低估變異數
  • 迴歸插補：用其他變數預測遺漏值。較精確但仍低估標準誤
  • 多重插補（MI）：產生多組插補資料集，各自分析後合併結果。最推薦的方法
  • EM演算法：Maximum Likelihood 基礎的迭代方法，適用於 MAR

Imputation Methods

Replace missing values with estimated values to retain full sample size.

  • Mean substitution: Replace with the variable mean. Simple but underestimates variance
  • Regression imputation: Predict missing values from other variables. More accurate but still underestimates SE
  • Multiple imputation (MI): Generate multiple imputed datasets, analyze each, then pool results. Most recommended
  • EM algorithm: Maximum Likelihood iterative method, appropriate for MAR

經驗法則：遺漏值處理

  • 遺漏比例 < 5%：通常可安全使用完整刪除法
  • 5%-15%：需要進一步診斷遺漏機制，考慮插補
  • > 15%：必須認真處理，多重插補或 EM 方法為佳
  • 單一變數遺漏 > 50%：考慮直接刪除該變數
  • Missing < 5%: Generally safe to use listwise deletion
  • 5%-15%: Need to diagnose the missing mechanism; consider imputation
  • > 15%: Must be handled seriously; MI or EM methods preferred
  • Single variable > 50% missing: Consider dropping that variable entirely

二、離群值（Outliers）

離群值是指與其他觀察值有顯著差異的極端值。離群值可能是資料輸入錯誤，也可能是真實但罕見的觀察。關鍵問題是：這些極端值會扭曲統計結果。

Outliers are observations that are markedly different from other cases. They may be data entry errors or genuine but rare observations. The key issue: extreme values can distort statistical results.

單變量離群值偵測

Z-score 方法：將每個值標準化，檢查是否超出門檻。

z = (X - X̄) / s X̄ = 平均數, s = 標準差

判斷門檻：

  • 小樣本 (n < 80)：|z| > 2.5 為離群值
  • 大樣本 (n ≥ 80)：|z| > 3.0 或 4.0

箱型圖法：超出 Q1 - 1.5*IQR 或 Q3 + 1.5*IQR 為離群值。

Z-score standardizes each value; box plot uses interquartile range (IQR).

Z-score method: Standardize each value and check against thresholds.

z = (X - X̄) / s X̄ = mean, s = standard deviation

Thresholds:

  • Small samples (n < 80): |z| > 2.5 is an outlier
  • Large samples (n ≥ 80): |z| > 3.0 or 4.0

Box plot method: Values beyond Q1 - 1.5*IQR or Q3 + 1.5*IQR are outliers.

多變量離群值偵測

Mahalanobis D² 距離：衡量每個觀察值與所有變數聯合中心（centroid）的距離，同時考慮變數間的相關性。

D² = (x - x̄)' S-1 (x - x̄) S-1 = 共變異數矩陣的反矩陣

判斷方式：以卡方分佈檢定 D² 值，自由度 = 變數數，通常以 p < .001 為門檻。

Mahalanobis D² measures distance from the centroid, accounting for variable correlations.

Mahalanobis D² distance: Measures each observation's distance from the centroid of all variables, accounting for correlations between them.

D² = (x - x̄)' S-1 (x - x̄) S-1 = inverse of the covariance matrix

Assessment: Test D² against chi-square distribution with df = number of variables; typically p < .001 threshold.

離群值的處理策略

  • 保留：如果離群值代表真實的極端情況（如罕見疾病、超高收入者），且研究目的就是了解這種現象
  • 刪除：如果確認是資料輸入錯誤，或該觀察明顯不屬於目標母群
  • 修正（Winsorize）：將極端值替換為較接近但仍處於極端的值（如替換為第95百分位數的值）
  • 分開分析：分別對有/無離群值的資料進行分析，比較結果是否一致
  • Retain: If the outlier represents a genuine extreme case (rare disease, ultra-high income) relevant to the research purpose
  • Delete: If confirmed as a data entry error, or the case clearly does not belong to the target population
  • Winsorize: Replace extreme values with less extreme but still high/low values (e.g., 95th percentile value)
  • Run separate analyses: Analyze with and without outliers to compare if results are consistent

Z-score 離群值偵測計算器

輸入觀察值、平均數和標準差來計算z-score，並判斷是否為離群值。

Enter the observed value, mean, and standard deviation to compute the z-score and determine if it is an outlier.

三、常態性檢驗（Normality Testing）

大多數多變量統計方法都假設資料服從常態分佈。違反常態性假設會影響統計檢定的結果，尤其在小樣本中更為嚴重。常態性可從兩個面向來檢驗：偏態和峰度。

Most multivariate methods assume data follow a normal distribution. Violating normality affects statistical test results, especially severely in small samples. Normality is assessed along two dimensions: skewness and kurtosis.

偏態（Skewness）

衡量分佈的對稱性。偏態 = 0 為完美對稱。

Skewness = [n / (n-1)(n-2)] × Σ[(Xi - X̄) / s]³ 正偏 = 右尾較長 | 負偏 = 左尾較長

Positive skew: right tail is longer; Negative skew: left tail is longer.

Measures the symmetry of a distribution. Skewness = 0 is perfectly symmetric.

Skewness = [n / (n-1)(n-2)] × Σ[(Xi - X̄) / s]³ Positive = right tail longer | Negative = left tail longer

峰度（Kurtosis）

衡量分佈的尖峭程度。峰度 = 0 （超額峰度）為正常。

Kurtosis = {[n(n+1) / (n-1)(n-2)(n-3)] × Σ[(Xi - X̄) / s]&sup4;} - [3(n-1)² / (n-2)(n-3)] 正峰度 = 尖峭、重尾 | 負峰度 = 平坦、輕尾

Positive kurtosis (leptokurtic): peaked, heavy tails; Negative (platykurtic): flat, light tails.

Measures the peakedness of a distribution. Excess kurtosis = 0 for a normal distribution.

Kurtosis = {[n(n+1) / (n-1)(n-2)(n-3)] × Σ[(Xi - X̄) / s]&sup4;} - [3(n-1)² / (n-2)(n-3)] Positive (leptokurtic): peaked, heavy tails | Negative (platykurtic): flat, light tails

常態性判斷標準

指標 可接受範圍（小樣本） 可接受範圍（大樣本） 說明
偏態 Skewness ± 1.0 ± 2.0 絕對值越接近0越好
峰度 Kurtosis ± 3.0 ± 7.0 峰度對常態性的影響比偏態更嚴重
Shapiro-Wilk p > .05 表示未顯著偏離常態 適用於 n < 2000
Kolmogorov-Smirnov p > .05 表示未顯著偏離常態 較保守

視覺化檢驗：Q-Q圖

Q-Q圖（Quantile-Quantile Plot）是比較觀察資料分位數與理論常態分佈分位數的散佈圖。如果資料服從常態分佈，點應該大致落在45度對角線上。系統性偏離對角線表示非常態。

  • 點在兩端彎曲向上 → 正偏態（右偏）
  • 點在兩端彎曲向下 → 負偏態（左偏）
  • S形彎曲 → 峰度問題

A Q-Q plot (Quantile-Quantile Plot) compares observed data quantiles with theoretical normal distribution quantiles. If data are normally distributed, points should fall approximately along the 45-degree diagonal line. Systematic departures indicate non-normality.

  • Points curve upward at both ends → positive skew
  • Points curve downward at both ends → negative skew
  • S-shaped curve → kurtosis issues

四、線性與同質性

除了常態性，多變量分析還有兩個重要假設：線性關係（linearity）和同變異數性/等分散性（homoscedasticity）。

Beyond normality, multivariate analysis has two other important assumptions: linearity and homoscedasticity.

線性（Linearity）

迴歸模型假設依變數和自變數之間存在直線關係。如果真實關係是曲線的，線性模型會低估真正的關聯程度。

檢驗方式：

  • 檢查殘差圖（residual plots）——殘差應隨機分佈
  • 偏迴歸圖（partial regression plots）——應呈現直線
  • 加入二次項或交互項並檢驗其顯著性

Linearity: check residual plots for random scatter; non-random patterns suggest nonlinearity.

Regression models assume a straight-line relationship between DV and IVs. If the true relationship is curvilinear, linear models will underestimate the actual association.

Testing:

  • Examine residual plots — residuals should scatter randomly
  • Partial regression plots — should show linear trend
  • Add quadratic or interaction terms and test significance

同變異數性（Homoscedasticity）

依變數的變異程度在自變數的所有水準上大致相等。違反時稱為異質性（heteroscedasticity）。

檢驗方式：

  • 殘差圖：殘差的散佈幅度不應隨預測值增大而改變（不應呈扇形）
  • Levene's test：檢驗各組變異數是否相等（p > .05 → 同質）
  • Breusch-Pagan test（迴歸中使用）

Homoscedasticity: equal variance of DV across all levels of IVs; Levene's test checks this.

The DV exhibits equal levels of variance across all levels of the IV. Violation is called heteroscedasticity.

Testing:

  • Residual plots: spread of residuals should not change with predicted values (no fan shape)
  • Levene's test: Tests equality of variances across groups (p > .05 → homogeneous)
  • Breusch-Pagan test (for regression)

違反假設的後果

  • 非常態：標準誤估計不準確 → 統計檢定（p值）不可靠
  • 非線性：低估真實的變數關聯程度 → 模型解釋力不足
  • 異質性：迴歸的標準誤偏誤 → 信賴區間不準確
  • 好消息：當樣本量夠大（n > 200），由中央極限定理，違反常態性的影響會大幅降低
  • Non-normality: Inaccurate SE estimates → unreliable statistical tests (p-values)
  • Non-linearity: Underestimate true variable associations → insufficient model explanatory power
  • Heteroscedasticity: Biased regression SEs → inaccurate confidence intervals
  • Good news: With large samples (n > 200), the Central Limit Theorem substantially reduces the impact of non-normality

五、資料轉換（Data Transformations）

當資料違反常態性或同質性假設時，可以透過數學轉換來改善資料的分佈形狀。轉換後的變數取代原始變數進行分析。

When data violate normality or homoscedasticity assumptions, mathematical transformations can improve the distribution shape. Transformed variables replace originals in the analysis.

轉換方法 公式 適用情況 效果
對數轉換 X' = log(X) or ln(X) 正偏態（中度到重度） 壓縮右尾，最常用
平方根轉換 X' = √X 正偏態（輕度到中度） 效果比對數溫和
倒數轉換 X' = 1/X 正偏態（極度嚴重） 最強的壓縮效果
平方轉換 X' = X² 負偏態 擴展右尾以修正左偏
Box-Cox 轉換 X' = (Xλ - 1) / λ 自動找最佳 λ 值 通用族

轉換的注意事項

  • 轉換會改變變數的解釋方式——例如對數轉換後的迴歸係數代表「百分比變化」而非「單位變化」
  • 包含零或負值的變數需先加常數（如 log(X + 1)）
  • 倒數轉換會反轉排序方向
  • 先嘗試較溫和的轉換（平方根），再考慮較強的轉換（對數、倒數）
  • 轉換不是萬能的——有時候應考慮使用無母數方法或穩健估計
  • Transformations change variable interpretation — e.g., log-transformed regression coefficients represent "percentage change" not "unit change"
  • Variables with zeros or negatives need a constant added first (e.g., log(X + 1))
  • Inverse transformation reverses the rank order
  • Try milder transformations (square root) before stronger ones (log, inverse)
  • Transformations are not always the answer — consider nonparametric methods or robust estimation

七、關鍵名詞解釋

MCAR / MAR / MNAR

MCAR（Missing Completely At Random）：遺漏完全隨機，與任何變數無關。MAR（Missing At Random）：遺漏與其他已觀察到的變數有關。MNAR（Missing Not At Random）：遺漏與未觀察到的值（即遺漏值本身）有關。這三個概念決定了適合使用的遺漏值處理方法。

MCAR: Missingness is completely unrelated to any variable. MAR: Missingness is related to other observed variables. MNAR: Missingness is related to the unobserved (missing) value itself. These three concepts determine the appropriate missing data handling method.

Mahalanobis D² 距離

衡量某個觀察值在多變量空間中與所有觀察值中心（centroid）的距離。不同於歐式距離，D² 會考慮變數之間的共變異數（相關性），因此可以偵測到多變量的異常組合模式。以卡方分佈進行顯著性檢定，自由度等於變數個數。

Measures the distance of an observation from the centroid of all observations in multivariate space. Unlike Euclidean distance, D² accounts for covariance (correlations) between variables, enabling detection of unusual multivariate combination patterns. Tested for significance using the chi-square distribution with df = number of variables.

多重插補（Multiple Imputation）

處理遺漏值最推薦的方法。流程：(1) 產生 m 組（通常5-20組）包含不同插補值的完整資料集；(2) 分別對每組資料進行分析；(3) 使用 Rubin's rules 合併結果。優於單一插補的原因：它反映了插補值的不確定性，產生更準確的標準誤和信賴區間。

The most recommended method for handling missing data. Process: (1) Generate m (typically 5-20) complete datasets with different imputed values; (2) Analyze each dataset separately; (3) Pool results using Rubin's rules. Superior to single imputation because it reflects the uncertainty of imputed values, producing more accurate SEs and confidence intervals.

同變異數性（Homoscedasticity）

依變數的變異程度在自變數的所有水準上大致相等的假設。違反（稱為異質性 heteroscedasticity）會導致迴歸係數的標準誤估計偏誤，使得假設檢定和信賴區間不可靠。檢驗方式包括殘差圖目視檢查和 Levene's test。

The assumption that the DV exhibits equal levels of variance across all levels of the IV. Violation (heteroscedasticity) biases standard error estimates for regression coefficients, making hypothesis tests and confidence intervals unreliable. Tested via visual inspection of residual plots and Levene's test.

Box-Cox 轉換

一種通用的冪次轉換族，透過最大概似估計法自動尋找最佳的 λ 參數值。特殊情況：λ = 1 表示不需要轉換；λ = 0.5 相當於平方根轉換；λ = 0 相當於對數轉換；λ = -1 相當於倒數轉換。適用於正值的連續變數。

A general family of power transformations that uses maximum likelihood to automatically find the optimal λ parameter. Special cases: λ = 1 means no transformation needed; λ = 0.5 equals square root; λ = 0 equals log; λ = -1 equals inverse. Applicable to positive continuous variables.
