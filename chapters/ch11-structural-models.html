Ch.11 結構方程模型檢驗

從測量模型到結構模型：檢驗理論假說的完整流程

1. 從測量模型到結構模型

白話解釋

測量模型確認工具可靠，結構模型檢驗理論假設。就像蓋房子：測量模型是檢查建材（磚頭、鋼筋）品質是否合格；結構模型是檢驗房子的設計圖（哪根柱子撐哪面牆）是否正確。先確認工具沒問題，才能放心用它來驗證你的理論。

兩階段法 (Anderson & Gerbing, 1988)

階段 目的 檢驗內容
第一階段：測量模型 (CFA) 確認測量工具的信效度 因素負荷量、CR、AVE、區辨效度
第二階段：結構模型 檢驗變數間的因果關係 路徑係數、顯著性、R²、模型適配
SEM 包含兩個部分：測量模型（藍色區域）確認指標與建構的關係；結構模型（黃色區域）檢驗建構之間的路徑假說

2. 結構模型設定

模型設定的三個關鍵要素

  • 假說路徑 (Hypothesized Paths)：根據理論決定哪些建構之間存在因果關係
  • 路徑方向 (Path Direction)：箭頭從自變數指向依變數
  • 限制條件 (Constraints)：某些路徑可設定為零（即不存在影響）

等價模型問題

一組資料可能可以被多個不同的結構模型同樣良好地解釋（等價模型）。例如 A→B 和 B→A 可能產生完全相同的適配度指標。因此，SEM 不能「證明」因果關係，只能說「資料與理論模型一致」。研究者必須根據理論而非僅靠統計來決定路徑方向。

外生 vs 內生建構

類型 角色 特徵
外生建構 (Exogenous) 只有箭頭「射出」 類似迴歸的自變數，不被模型中其他變數解釋
內生建構 (Endogenous) 有箭頭「射入」 類似迴歸的依變數，有對應的 R² 值

3. 結構模型評估

結構模型的評估主要關注三個面向：路徑係數的大小與方向、路徑是否統計顯著、以及模型整體的解釋力。

路徑係數 (β) 與顯著性檢定

白話解釋

路徑係數就像迴歸的標準化係數 (beta)，告訴你「當自變數增加一個標準差，依變數會變化幾個標準差」。β = 0.45 代表自變數每增加一個標準差，依變數平均增加 0.45 個標準差。而 t 值或 z 值則告訴你這個影響是「真的」還是「偶然的」。

指標 門檻 意義
β -1 ~ +1 標準化影響強度與方向
|t| > 1.96 (p < .05) 雙尾檢定 95% 信心水準
p < 0.05 路徑係數統計上顯著

R² 判斷準則

R² 代表內生建構被所有自變數共同解釋的變異比例。

R² = 1 - (SSresidual / SStotal) 0.75 = 強、0.50 = 中等、0.25 = 弱 (Hair et al., 2019)

注意：R² 的解釋取決於研究領域。在消費者行為研究中，R² = 0.20 可能就已算合理；在工程研究中可能需要 R² > 0.90。

4. 模型比較

巢套模型的 Δχ² 差異檢定

當兩個模型存在巢套關係（一個是另一個的特例，增加或移除限制條件），可用卡方差異檢定來比較。

Δχ² = χ²constrained - χ²free Δdf = dfconstrained - dffree

白話解釋

限制模型就是「綁手綁腳」的模型（多了限制條件），自由模型就是「自由發揮」的模型。如果綁手綁腳後適配度顯著變差（Δχ² 顯著），代表那個被限制的路徑確實重要，不應該移除。

非巢套模型比較：AIC / BIC

當兩個模型不存在巢套關係時，不能使用 Δχ²，改用資訊準則來比較。

準則 原則 特性
AIC 值越小越好 兼顧適配度與簡約性；適合預測導向
BIC 值越小越好 對參數數量懲罰更重；偏好簡約模型

競爭模型策略

最佳實踐是提出多個理論上合理的替代模型，而非僅呈現單一模型。這能增強研究的說服力。

  • 假說模型 (Hypothesized Model)：根據文獻與理論提出的主要模型
  • 替代模型 (Alternative Model)：改變路徑方向或加減路徑的版本
  • 簡約模型 (Parsimonious Model)：移除不顯著路徑後的精簡版本

5. 模型修正策略

理論驅動修正

  • 根據理論基礎增加或刪除路徑
  • 修正前需有文獻支持
  • 需要在新樣本上進行交叉驗證

推薦做法：修正有理論依據，結論可推廣

資料驅動修正

  • 根據修正指標 (Modification Indices, MI) 調整
  • MI 告訴你「如果加這條路徑，χ² 可以降多少」
  • 有過度適配 (overfitting) 的風險

風險：純粹追求統計改善，結果可能不可複製

規範搜尋的風險

反覆嘗試不同的路徑組合以尋找「最佳」模型（specification search），會嚴重膨脹 Type I 錯誤率，使偶然發現的關係被誤判為真實效果。每次修正都應有理論理由，且應報告所有嘗試過的模型。

6. 報告撰寫

SEM 結果報告檢核清單

報告項目 具體內容 位置
樣本資訊 樣本大小、抽樣方法、回收率 方法
估計方法 ML、MLR、WLSMV 等 方法
測量模型適配 χ²(df)、CFI、TLI、RMSEA [90% CI]、SRMR 結果
信效度 CR、AVE、HTMT 或 Fornell-Larcker 結果
結構模型適配 同上的適配指標組合 結果
路徑係數 標準化 β、SE、t/z 值、p 值（或 95% CI） 結果
R² 每個內生建構的 R² 值 結果
假說表 假說內容、路徑、β、p、結論（支持/不支持） 結果

報告範例用語

「結構模型的適配度良好 (χ²(52) = 78.34, p = .011; CFI = .97; TLI = .96; RMSEA = .045 [.021, .066]; SRMR = .038)。服務品質對顧客滿意度有顯著正向影響 (β = .58, p < .001)，滿意度對忠誠度亦有顯著正向影響 (β = .67, p < .001)。滿意度的 R² = .34，忠誠度的 R² = .45，顯示模型具有中等解釋力。」

7. 自我測驗

測試你對結構模型的理解

Q1：Anderson & Gerbing (1988) 的兩階段法中，為什麼要先評估測量模型再評估結構模型？

因為測量模型比較簡單，先做容易的
因為如果測量工具有問題，建構之間的路徑關係就不可信
因為軟體要求必須先跑 CFA
正確！測量模型確保你的測量工具是可靠且有效的。如果連測量都有問題（信效度不足），那基於這些測量所建立的結構路徑就失去了意義。

Q2：兩個非巢套的模型要進行比較，應使用什麼方法？

Δχ² 差異檢定
AIC 或 BIC 比較（值越小越好）
直接比較 R² 即可
正確！Δχ² 只適用於巢套模型。非巢套模型應使用 AIC 或 BIC 等資訊準則，這些準則同時考慮模型適配度與簡約性。

Q3：一位研究者根據修正指標 (MI) 連續加了 5 條未預期的路徑，使 CFI 從 .91 提升到 .97。這個做法有什麼問題？

完全沒問題，適配度越好越好
只需要加兩條路徑就夠了
這是資料驅動修正，有過度適配的風險，且缺乏理論支持的路徑結果不可複製
正確！僅根據修正指標修正模型是典型的「規範搜尋」，會嚴重膨脹 Type I 錯誤率。每一次修正都必須有理論依據，而不是單純追求更好的適配度數字。

Q4：報告 SEM 結果時，以下哪項是最不可或缺的？

只報告 CFI 和 RMSEA 就夠了
適配指標組合、標準化路徑係數（含 SE 與 p 值）、以及 R²
只需報告顯著的路徑
正確！完整的 SEM 報告應包含多個適配指標、所有路徑係數（包括不顯著的）及其標準誤和 p 值、以及所有內生建構的 R²。選擇性報告只有顯著結果是不良的研究實踐。
