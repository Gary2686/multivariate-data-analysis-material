驗證性因素分析

Ch.10 CFA 模型設定、識別、適配度評估與模型修正

一、CFA 概述

驗證性因素分析（Confirmatory Factor Analysis, CFA）是 SEM 的測量模型部分，用來檢驗研究者事先提出的測量理論——即潛在建構與其觀察指標之間的對應關係——是否與實際資料吻合。CFA 是從 EFA 的「探索」進入「驗證」階段的關鍵方法。

Confirmatory Factor Analysis (CFA) is the measurement model component of SEM, used to test whether a researcher's pre-specified measurement theory — the correspondence between latent constructs and their observed indicators — is consistent with actual data. CFA is the critical method for moving from EFA's "exploration" to "confirmation."

白話解釋

CFA 就像考試前先確認考卷題目真的在考你想考的東西——你設計了一份問卷來測量「工作滿意度」，但這些題目真的都在量同一個東西嗎？有沒有哪題其實在量別的概念？CFA 幫你用統計方法回答這些問題。

CFA is like confirming that test items actually measure what you intend before the exam — you designed a questionnaire for "job satisfaction," but do all items truly measure the same thing? Is any item actually measuring a different concept? CFA helps answer these questions statistically.

面向 EFA CFA
理論需求 不需要事先理論 必須有明確的測量理論
因素數量 由資料決定 事先指定
交叉負荷 全部自由估計 固定為零
適配度評估 有限 全面（多種適配度指標）
使用時機 量表開發初期、探索因素結構 量表驗證、已有理論基礎時

二、CFA 模型設定

CFA 模型設定是將測量理論轉化為可估計的統計模型。研究者必須明確指定哪些觀察變數（指標）對應哪個潛在建構，以及哪些路徑被約束（固定）為零。

CFA model specification translates measurement theory into an estimable statistical model. Researchers must explicitly specify which observed variables (indicators) correspond to which latent constructs, and which paths are constrained (fixed) to zero.

圖：單因素 CFA 模型 — 一個潛在建構由五個觀察指標測量

CFA 中需要指定的元素

  1. 潛在建構（因素）：數量和名稱
  2. 指標歸屬：每個觀察變數指定到一個建構
  3. 因素負荷量：自由估計（歸屬路徑）或固定為零（交叉負荷）
  4. 誤差項：每個指標各自的誤差變異
  5. 因素間相關：允許建構之間的共變異
  1. Latent constructs (factors): Number and names
  2. Indicator assignments: Each observed variable assigned to one construct
  3. Factor loadings: Freely estimated (assigned paths) or fixed to zero (cross-loadings)
  4. Error terms: Unique error variance for each indicator
  5. Factor correlations: Allowing covariance between constructs

設定規則

  • 每個指標只能歸屬一個建構
  • 非歸屬路徑的負荷量固定為 0
  • 誤差項之間不應有共變異（同源模型要求）
  • 每個建構至少需要 3 個指標（理想 ≥ 4 個）
  • 需設定標記變數或固定因素變異數以確立尺度
  • Each indicator loads on only one construct
  • Non-assigned loadings fixed to 0
  • Error terms should have no covariance (congeneric model)
  • Each construct requires at least 3 indicators (ideally ≥ 4)
  • A marker variable or fixed factor variance must be set for scaling

三、CFA 模型識別

CFA 模型識別決定模型是否能產生唯一的參數估計值。識別不足的模型無法求解，恰好識別的模型無法評估適配度。

CFA model identification determines whether a model can produce unique parameter estimates. Under-identified models cannot be solved; just-identified models cannot assess fit.

df = p(p+1)/2 - q
p = 觀察變數總數；q = 自由估計的參數數量

三指標規則（Three-Indicator Rule）

每個潛在建構至少需要三個指標才能達到恰好識別（df = 0）。這是因為：

  • 3 個指標提供 3(4)/2 = 6 個唯一資訊（共變異+變異數）
  • 需估計：2 個負荷量（1個固定為標記變數）+ 1 個因素變異 + 3 個誤差變異 = 6 個參數
  • df = 6 - 6 = 0（恰好識別）
  • 4 個指標時：10 - 8 = 2（過度識別，可評估適配度）

Each latent construct needs at least three indicators for just-identification (df = 0). This is because:

  • 3 indicators provide 3(4)/2 = 6 unique pieces of information (covariances + variances)
  • Parameters to estimate: 2 loadings (1 fixed as marker) + 1 factor variance + 3 error variances = 6
  • df = 6 - 6 = 0 (just-identified)
  • With 4 indicators: 10 - 8 = 2 (over-identified, fit can be assessed)

標記變數與尺度設定

潛在變數沒有自然的測量單位，因此需要設定尺度。兩種常見方法：

  • 標記變數法：將一個指標的負荷量固定為 1.0，潛在變數的單位就等同於該指標的單位。通常選擇與建構定義最相關的指標。
  • 因素變異數法：固定潛在變數的變異數為 1.0，產生標準化解。所有負荷量都可自由估計。

Latent variables have no natural measurement units, so scaling must be set. Two common approaches:

  • Marker variable: Fix one indicator's loading to 1.0; the latent variable adopts that indicator's units. Usually choose the indicator most representative of the construct.
  • Factor variance: Fix the latent variable's variance to 1.0, producing a standardized solution. All loadings are freely estimated.

四、CFA 適配度評估

CFA 使用與 SEM 相同的適配度指標體系。目標是確認觀察到的共變異矩陣與模型所隱含的共變異矩陣之間的差異是否在可接受範圍內。

CFA uses the same fit index framework as SEM. The goal is to confirm that discrepancies between observed and model-implied covariance matrices are within acceptable bounds.

指標 良好適配 可接受 CFA 特別注意
χ²/df < 3.0 < 5.0 CFA 中常因大樣本而顯著
CFI ≥ .95 ≥ .90 CFA 中最常報告的指標
TLI ≥ .95 ≥ .90 與 CFI 搭配使用
RMSEA ≤ .06 ≤ .08 報告 90% CI
SRMR ≤ .08 ≤ .10 測量模型的殘差指標

適配度不佳時的警訊

  • CFI < .90 或 RMSEA > .10：模型需要重大修改或重新概念化
  • 個別指標的標準化殘差 > |4.0|：該指標可能有問題
  • 某指標的標準化負荷量 < .50：考慮刪除
  • 模型無法收斂（non-convergence）：通常表示模型設定有誤或樣本量不足
  • Heywood case（負誤差變異或負荷量 > 1.0）：嚴重的模型問題
  • CFI < .90 or RMSEA > .10: Model needs major revision or reconceptualization
  • Individual indicator standardized residual > |4.0|: That indicator may be problematic
  • An indicator's standardized loading < .50: Consider deletion
  • Non-convergence: Usually indicates specification error or insufficient sample size
  • Heywood case (negative error variance or loading > 1.0): Serious model problems

五、模型修正

當 CFA 模型適配度不佳時，可能需要對模型進行修正。修正的依據來自兩個主要的診斷工具：修正指標（MI）和標準化殘差。但所有修正都必須有理論支持。

When CFA model fit is inadequate, modification may be needed. Modifications are guided by two primary diagnostic tools: modification indices (MI) and standardized residuals. All modifications must have theoretical justification.

診斷工具 作用 判斷標準 注意事項
修正指標 (MI) 預估釋放某路徑後 χ² 的減少量 MI ≥ 4.0 表示有意義 必須有理論支持
標準化殘差 觀察與預測共變異的差異 < |2.50| 可接受；> |4.0| 需處理 注意模式而非個別值
期望參數改變 (EPC) 預估釋放路徑後參數值 搭配 MI 一起判斷 大 EPC + 小 MI 可能假陽性

模型修正的黃金法則

  1. 每次只做一個修改，然後重新估計模型
  2. 修改必須有理論上的合理性
  3. 刪除題項前先確認不會違反三指標規則
  4. 若修改超過20%的題項，應使用新的資料集驗證
  5. 允許誤差共變異應有實質理由（如共用方法、相似措辭）
  6. 所有修改必須在論文中明確報告
  1. Make one modification at a time, then re-estimate
  2. Modifications must have theoretical justification
  3. Before dropping items, ensure the three-indicator rule is not violated
  4. If more than 20% of items are changed, validate with a new dataset
  5. Error covariances should have substantive reasons (e.g., shared method, similar wording)
  6. All modifications must be explicitly reported in publications

七、進階議題

一階 CFA（First-Order CFA）

標準的 CFA 模型，潛在建構直接由觀察指標測量。最常見的形式。

  • 每個建構直接連結到其指標
  • 建構之間可以自由相關
  • 適合當各建構之間不存在更高層次的共同因素時
  • 範例：工作滿意度、組織承諾、離職意圖——三個獨立但相關的建構

First-Order CFA

The standard CFA model where latent constructs are directly measured by observed indicators. The most common form.

  • Each construct directly links to its indicators
  • Constructs can freely correlate
  • Appropriate when no higher-order common factor underlies the constructs
  • Example: Job satisfaction, organizational commitment, turnover intention — three independent but correlated constructs

二階 CFA（Second-Order CFA）

當多個一階因素被假設為某個更高階、更抽象建構的子面向時使用。

  • 二階因素解釋一階因素之間的共變異
  • 一階因素變成二階因素的「指標」
  • 比一階模型更簡約（fewer parameters）
  • 識別條件：至少需要 3 個一階因素
  • 範例：「智力」（二階）由語文推理、數量推理、空間推理（一階）組成

比較：若二階模型的適配度與一階模型相當（Δχ² 不顯著），支持使用更簡約的二階模型。

Second-Order CFA

Used when multiple first-order factors are hypothesized as sub-dimensions of a higher-order, more abstract construct.

  • The second-order factor explains covariances among first-order factors
  • First-order factors become "indicators" of the second-order factor
  • More parsimonious than first-order models (fewer parameters)
  • Identification: At least 3 first-order factors required
  • Example: "Intelligence" (second-order) composed of verbal, quantitative, and spatial reasoning (first-order)

Comparison: If second-order fit is comparable to first-order (Δχ² non-significant), the more parsimonious second-order model is supported.

題目打包法（Parceling）

打包（Parceling）是將多個題項加總或平均為「包裹」（parcels）來作為指標，而非使用個別題項。

  • 優點：減少估計參數、改善常態性、提高信度
  • 缺點：可能遮蔽因素結構問題、交叉負荷無法偵測
  • 使用條件：一階因素結構已透過 EFA/CFA 確認為單因素
  • 方法：隨機分配法、因素負荷量平衡法
  • 爭議：學界對打包的適用性仍有不同意見；建議在題項較多且結構明確時才使用

Parceling involves summing or averaging multiple items into "parcels" as indicators, rather than using individual items.

  • Advantages: Fewer parameters, improved normality, higher reliability
  • Disadvantages: May mask factor structure issues; cross-loadings undetectable
  • Condition: First-order structure confirmed as unidimensional via EFA/CFA
  • Methods: Random assignment, factor-loading balanced approach
  • Controversy: Ongoing debate about appropriateness; recommended only with many items and clear structure
測量恆等性基礎（Measurement Invariance）

測量恆等性檢驗同一份量表在不同群組（如性別、國家）中是否以相同方式測量相同建構。這是跨群組比較的前提。

測量恆等性的層級（由寬鬆到嚴格）：

  1. 構形恆等性（Configural）：相同的因素結構適用於所有群組
  2. 計量恆等性（Metric）：因素負荷量在群組間相等
  3. 純量恆等性（Scalar）：截距在群組間相等（比較潛在平均數的前提）
  4. 嚴格恆等性（Strict）：殘差變異在群組間相等

檢驗方法：透過多群組 CFA（Multi-Group CFA），依序對參數施加等值限制，比較不同限制程度下的模型適配度變化（ΔCFI ≤ .01 通常視為恆等性成立）。

Measurement invariance tests whether the same scale measures the same construct in the same way across different groups (e.g., gender, country). This is a prerequisite for cross-group comparisons.

Levels of measurement invariance (from lenient to strict):

  1. Configural: Same factor structure holds across all groups
  2. Metric: Factor loadings equal across groups
  3. Scalar: Intercepts equal across groups (prerequisite for latent mean comparisons)
  4. Strict: Residual variances equal across groups

Testing method: Through Multi-Group CFA, sequentially constrain parameters to equality and compare fit changes (ΔCFI ≤ .01 typically indicates invariance holds).
