Ch.3 探索性因素分析

資料縮減與潛在結構辨識

一、EFA 簡介與目的

探索性因素分析（EFA）是一種多變量統計技術，用於從大量觀察變數中辨識潛在的因素結構。其核心目標有二：

  • 資料縮減（Data Reduction）：將大量相關變數濃縮為少數具代表性的因素
  • 結構探索（Structure Discovery）：發現變數之間隱含的分群模式

EFA 屬於相依技術（interdependence technique），不區分自變數與依變數，而是同時分析所有變數間的關係。

Exploratory Factor Analysis (EFA) is a multivariate technique used to identify latent factor structures from a large set of observed variables. Its two core goals are:

  • Data Reduction: Condense many correlated variables into a smaller set of representative factors
  • Structure Discovery: Uncover hidden grouping patterns among variables

EFA is an interdependence technique -- it does not distinguish independent from dependent variables but analyzes all interrelationships simultaneously.

白話解釋：EFA 就像整理衣櫃

想像你有一個凌亂的衣櫃，裡面有 50 件衣物散落各處。EFA 就像是幫你把衣物分類：上衣歸一堆、褲子歸一堆、外套歸一堆。

每一「堆」就是一個因素（factor），而每件衣物屬於哪一堆的程度就是因素負荷量（factor loading）。有些衣物（例如可當外套也可當上衣的罩衫）可能同時跟兩堆有關，這就是交叉負荷（cross-loading）的情況。

Imagine you have a messy closet with 50 items of clothing scattered everywhere. EFA is like sorting them into categories: tops in one pile, pants in another, jackets in a third.

Each "pile" is a factor, and how strongly each item belongs to a pile is its factor loading. Some items (like a jacket-shirt hybrid) may relate to two piles at once -- that is a cross-loading situation.

圖：EFA 將多個觀察變數縮減為少數潛在因素

二、適合度檢驗

在執行 EFA 之前，必須先確認資料是否適合進行因素分析。如果變數之間相關太低，就無法萃取有意義的因素。

Before running EFA, you must verify that the data is suitable for factor analysis. If correlations among variables are too low, meaningful factors cannot be extracted.

KMO 取樣適切性量數

KMO 衡量變數之間偏相關的程度，值介於 0 到 1 之間。偏相關越小，代表變數間的共同變異越多，越適合因素分析。

KMO measures the degree of partial correlations among variables, ranging from 0 to 1. Smaller partial correlations indicate more shared variance, making the data more suitable.

Bartlett 球形檢定

檢驗相關矩陣是否為單位矩陣（identity matrix）。若 p < .05，拒絕虛無假設，表示變數之間存在顯著相關，適合進行因素分析。

Tests whether the correlation matrix is an identity matrix. If p < .05, reject the null hypothesis -- significant correlations exist among variables, making factor analysis appropriate.

反映像相關矩陣

檢查對角線上的個別變數 MSA 值（Measure of Sampling Adequacy）。每個變數的 MSA 應 ≥ .50，若低於此標準，該變數應考慮刪除。

Examine the individual variable MSA values on the diagonal. Each variable's MSA should be ≥ .50; variables below this threshold should be considered for removal.

KMO 值 解釋
0.90+極佳（Marvelous）
0.80 - 0.89良好（Meritorious）
0.70 - 0.79中等（Middling）
0.60 - 0.69平庸（Mediocre）
0.50 - 0.59悲慘（Miserable）— 勉強可接受
< 0.50不可接受（Unacceptable）— 不應進行因素分析

KMO 公式

KMO = Σ Σ r2ij / (Σ Σ r2ij + Σ Σ a2ij) r = 簡單相關係數, a = 偏相關係數 (i ≠ j)

三、因素萃取方法

因素萃取的目的是從相關矩陣中找出最能代表變數間共同變異的因素。兩大類萃取方法的核心差異在於所分析的變異來源。

The goal of extraction is to find factors that best represent the shared variance among variables. The two major approaches differ in which variance they analyze.

主成分分析（Principal Component Analysis, PCA）

分析的是總變異（total variance），包含共同變異、獨特變異和誤差變異。對角線上放的是 1。

  • 目的：以最少數量的成分解釋最大比例的總變異
  • 適用時機：主要目標是資料縮減，或想保留所有變異資訊
  • 特色：萃取出的成分是觀察變數的線性組合，不假設潛在因素存在

Principal Component Analysis (PCA)

Analyzes total variance, including shared, unique, and error variance. Diagonal elements are set to 1.

  • Purpose: Explain maximum total variance with the fewest components
  • When to use: Primary goal is data reduction or preserving all variance information
  • Key feature: Components are linear combinations of observed variables; no latent factor assumption

共同因素分析（Common Factor Analysis）

只分析共同變異（common variance / communality），排除獨特變異與誤差。對角線上放的是共同性估計值（communality estimates）。

常見方法：

  • 主軸因子法（PAF, Principal Axis Factoring）：反覆估計共同性直到收斂
  • 最大概似法（ML, Maximum Likelihood）：提供適配度統計量，可進行模型檢定

適用時機：研究目標是辨識潛在因素結構，且理論上假設存在潛在因素

Common Factor Analysis

Analyzes only common variance (communality), excluding unique and error variance. Diagonal elements use communality estimates.

Common methods:

  • PAF (Principal Axis Factoring): Iteratively estimates communalities until convergence
  • ML (Maximum Likelihood): Provides goodness-of-fit statistics for model testing

When to use: Goal is identifying latent factor structures with a theoretical basis

決定因素數量的方法

  • 特徵值 > 1 準則（Kaiser Criterion）：保留特徵值大於 1 的因素。最常用但傾向高估因素數量
  • 陡坡圖（Scree Plot）：繪製特徵值遞減曲線，在曲線出現明顯「手肘」彎折處截斷
  • 累積解釋變異量：社會科學通常要求解釋 60% 以上的總變異
  • 平行分析（Parallel Analysis）：比較實際特徵值與隨機資料的特徵值，較為客觀
  • Eigenvalue > 1 Rule (Kaiser Criterion): Retain factors with eigenvalues exceeding 1. Most common but tends to overextract
  • Scree Plot: Plot eigenvalues in descending order; cut off where the curve shows a clear "elbow"
  • Cumulative Variance Explained: Social sciences typically require 60%+ of total variance explained
  • Parallel Analysis: Compare actual eigenvalues against those from random data; more objective

特徵值（Eigenvalue）的含義

特徵值 = 該因素所解釋之變異量的總和 = Σ λ²（該因素上所有負荷量的平方和） 特徵值 = 1 代表該因素僅等同於一個變數的資訊量

四、因素轉軸

未轉軸的因素解通常難以解釋。轉軸（rotation）的目的是使因素負荷量更加極端化（接近 0 或接近 1），從而使每個因素有更清楚的解釋意義。轉軸不會改變整體解釋變異量或共同性，只會重新分配各因素的解釋比例。

Unrotated solutions are typically hard to interpret. The purpose of rotation is to make factor loadings more extreme (closer to 0 or 1), giving each factor a clearer interpretation. Rotation does not change total variance explained or communalities -- it only redistributes variance across factors.

正交轉軸（Orthogonal Rotation）

假設因素之間互不相關（r = 0）。轉軸時保持因素軸之間 90 度直角。

  • Varimax：最常用。最大化每個因素內負荷量的變異，使每個因素有少數高負荷與多數低負荷的變數
  • Quartimax：最大化每個變數在各因素上負荷量的變異，傾向產生一個大的一般因素
  • Equamax：Varimax 與 Quartimax 的折衷

適用時機：理論上認為因素彼此獨立時使用

Orthogonal Rotation

Assumes factors are uncorrelated (r = 0). Maintains 90-degree angles between factor axes.

  • Varimax: Most common. Maximizes the variance of loadings within each factor, producing a few high and many low loadings per factor
  • Quartimax: Maximizes variance of loadings across factors for each variable; tends to produce one large general factor
  • Equamax: A compromise between Varimax and Quartimax

When to use: When theory suggests factors are independent of each other

斜交轉軸（Oblique Rotation）

允許因素之間存在相關。在社會科學中，潛在建構之間通常確實有相關，因此斜交轉軸往往更符合現實。

  • Promax：先進行 Varimax 正交轉軸，再允許因素相關。計算速度快
  • Direct Oblimin：直接最小化因素之間的交叉乘積。delta 參數控制相關程度

適用時機：理論上認為因素之間有關聯，或正交轉軸結果顯示因素間確有相關

注意：斜交轉軸會產生兩個矩陣——模式矩陣（Pattern Matrix）與結構矩陣（Structure Matrix）。解釋時以模式矩陣為主。

Oblique Rotation

Allows factors to be correlated. In social sciences, latent constructs are usually correlated, making oblique rotation more realistic.

  • Promax: Starts with Varimax then allows correlations. Computationally fast
  • Direct Oblimin: Directly minimizes cross-products between factors. Delta parameter controls correlation degree

When to use: When theory suggests factors are related, or orthogonal results show interfactor correlations

Note: Oblique rotation produces two matrices -- Pattern Matrix and Structure Matrix. Interpretation is based on the Pattern Matrix.

選擇轉軸法的經驗法則

Hair et al. 建議：先使用斜交轉軸。如果因素間相關低於 .30，則改用正交轉軸（因為兩者結果幾乎相同，而正交解較簡潔）。若相關高於 .30，則使用斜交轉軸結果。

Hair et al. suggest: start with oblique rotation. If interfactor correlations are below .30, switch to orthogonal (since results will be nearly identical, and orthogonal is simpler). If correlations exceed .30, use the oblique solution.

五、因素解釋與命名

因素解釋的核心工作是檢視因素負荷量矩陣，辨識每個因素上有顯著負荷的變數，並據此為因素命名。

The core task of interpretation is examining the factor loading matrix, identifying variables with significant loadings on each factor, and naming factors accordingly.

指標 門檻值 說明
顯著因素負荷量 |loading| ≥ .40 實務上通常以 .40 為最低標準；n ≥ 350 時 .30 亦可接受
交叉負荷 差距 ≥ .20 一個變數在兩個因素上的負荷差距應 ≥ .20，否則考慮刪除
共同性 h² ≥ .50 共同性低於 .50 的變數可能缺乏足夠的共同變異
每因素最少變數 ≥ 3 每個因素至少應有 3 個顯著負荷的變數才具穩定性

共同性（Communality）公式

h2i = Σ λ2ij h² = 變數 i 在所有保留因素上負荷量的平方和（即被因素解釋的變異比例）

因素命名的原則

  • 檢視每個因素上負荷量最高的變數，找出它們的共同主題
  • 因素名稱應反映理論概念，而非只是列出變數名稱
  • 注意負向負荷的變數——它們代表因素概念的反面
  • 負荷量越高的變數對因素命名的影響力越大
  • Examine the highest loading variables on each factor and identify their common theme
  • Factor names should reflect theoretical concepts, not just list variable names
  • Watch for negatively loaded variables -- they represent the opposite of the factor concept
  • Higher loading variables carry more influence on factor naming

六、因素分數與驗證

因素分析完成後，研究者通常需要為每個受試者計算因素分數（factor scores），以便在後續分析中使用。

After completing factor analysis, researchers typically need to compute factor scores for each respondent for use in subsequent analyses.

因素分數的計算方式

  • 總加量表法（Summated Scales）：將高負荷變數取平均值。最常用且最容易在不同樣本間複製
  • 迴歸法（Regression Method）：以因素負荷量為權重的加權線性組合。精確但難以跨樣本複製
  • Bartlett 法：以最小化獨特因素的影響為目標，只使用共同變異
  • Summated Scales: Average the high-loading variables. Most common and most replicable across samples
  • Regression Method: Weighted linear combination using factor loadings. Precise but hard to replicate
  • Bartlett Method: Minimizes unique factor influence; uses only common variance

因素結構的驗證

  • 分半驗證（Split-sample）：將樣本隨機分成兩半，分別進行 EFA，比較因素結構的一致性
  • 交叉驗證（Cross-validation）：在新的獨立樣本中驗證因素結構
  • CFA 驗證：使用驗證性因素分析（CFA）正式檢定 EFA 發現的因素結構
  • Split-sample: Randomly split the sample in half, run EFA on each, and compare factor structures
  • Cross-validation: Validate the factor structure on a new independent sample
  • CFA validation: Use confirmatory factor analysis to formally test the EFA-derived structure

互動計算器：共同性與解釋變異

共同性與變異量計算器

輸入一個變數在各因素上的因素負荷量（以空格或逗號分隔），計算該變數的共同性（h²）以及各因素的解釋變異量。

Enter a variable's factor loadings across factors (separated by spaces or commas) to compute communality (h²) and each factor's explained variance.
