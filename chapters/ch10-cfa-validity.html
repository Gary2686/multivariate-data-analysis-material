驗證性因素分析與建構效度

Ch.10 CFA 中的 AVE、CR、聚合效度與區辨效度之進階衡量

一、EFA 與 CFA 的差異

在前一章我們學到，探索性因素分析（EFA）是從資料中「探索」可能的因素結構。而驗證性因素分析（CFA）是用來「驗證」研究者事先提出的測量理論是否與實際資料吻合。

Previously we learned that Exploratory Factor Analysis (EFA) "explores" potential factor structures from data. Confirmatory Factor Analysis (CFA) "confirms" whether a pre-specified measurement theory fits the actual data.

特徵 EFA CFA
目的 探索因素結構 驗證（確認）測量理論
因素數量 由資料決定 由研究者事先指定
變數歸屬 所有變數在所有因素上都有負荷量 研究者指定每個變數只歸屬一個因素
交叉負荷 允許（自由估計） 固定為零（constrained）
理論基礎 不需要事先理論 必須有測量理論
適配度檢驗 有限 全面（χ², CFI, RMSEA 等）

心理計量學基礎

CFA 植根於心理計量學（Psychometrics）——心理學中專門研究如何正確衡量潛在心理建構的分支。心理計量原則要求：

  1. 只有理論上連結的題項-建構路徑才被自由估計
  2. 所有其他可能的負荷固定為零
  3. 殘差之間不存在共變異

符合這些條件的測量模型稱為同源模型（Congeneric Model），代表良好的心理計量屬性。

CFA is rooted in Psychometrics — the branch of psychology addressing proper quantitative measurement of latent constructs. Psychometric principles require:

  1. Only theoretically linked item-construct paths are freely estimated
  2. All other possible loadings are constrained to zero
  3. No covariance exists among residuals

A measurement model meeting these conditions is called a Congeneric Model, representing good psychometric properties.

二、CFA 測量模型的組成要素

指定 CFA 測量模型時，需要定義五個要素：

When specifying a CFA measurement model, five elements must be defined:

1. 潛在建構

以橢圓形表示的不可直接觀察的理論概念（如：工作滿意度、組織承諾）。

Unobservable theoretical concepts represented by ellipses/circles (e.g., job satisfaction, organizational commitment).

2. 觀察變數（指標）

以矩形表示的實際測量題項（如問卷問題）。

Actual measured items represented by rectangles (e.g., survey questions).

3. 因素負荷量模式

以單向箭頭表示建構指向指標的路徑（λ）。在 CFA 中，每個題項只指向一個建構。

Single-headed arrows from constructs to indicators (λ). In CFA, each item loads on only one construct.

4. 建構間關係

以雙向曲線箭頭表示建構之間的相關或共變異關係。

Two-headed curved arrows representing correlational or covariance relationships between constructs.

5. 誤差變異

每個指標都有一個誤差項（e），代表潛在建構未能解釋的變異量。

Each indicator has an error term (e) representing variance not explained by the latent construct.

圖：CFA 同源測量模型（Congeneric Model）

每個建構需要多少指標？

  • 最佳：每個建構 ≥ 4 個指標（過度識別，可計算適配度）
  • 可接受：3 個指標（恰好識別，df = 0，適配度完美但無意義）
  • 避免：2 個或更少的指標（不足識別，需額外約束條件）
  • Ideal: ≥ 4 indicators per construct (overidentified, fit can be assessed)
  • Acceptable: 3 indicators (just-identified, df = 0, perfect but meaningless fit)
  • Avoid: 2 or fewer indicators (underidentified, additional constraints needed)

三、CFA 中的建構效度評估

建構效度是 CFA/SEM 的主要目標之一。它評估一組觀察變數是否確實反映了其被設計來衡量的潛在建構。在 CFA 中，需要評估以下五個層面：

Construct validity is a primary objective of CFA/SEM. It assesses whether a set of measured variables actually reflects the theoretical constructs they are designed to measure. In CFA, five aspects are evaluated:

適配效度（Fit Validity）

CFA 的第一步是確認整體模型與資料的吻合程度。常用的適配度指標：

指標門檻值說明
χ²/df< 3.0 (寬鬆 < 5.0)卡方自由度比
CFI≥ .95 (可接受 ≥ .90)比較適配指標
RMSEA≤ .06 (可接受 ≤ .08)近似均方根誤差
SRMR≤ .08標準化均方根殘差

適配度達標後，才能進一步評估聚合效度與區辨效度。

Fit Validity

The first step in CFA is confirming overall model fit with the data. Common fit indices:

IndexThresholdDescription
χ²/df< 3.0 (lenient < 5.0)Chi-square to df ratio
CFI≥ .95 (acceptable ≥ .90)Comparative Fit Index
RMSEA≤ .06 (acceptable ≤ .08)Root Mean Square Error of Approximation
SRMR≤ .08Standardized Root Mean Square Residual

Only after acceptable fit can convergent and discriminant validity be assessed.

聚合效度（Convergent Validity）在 CFA 中的評估

確認同一建構的指標們「聚合」在一起，共享高比例的共同變異。三個評估工具：

工具一：因素負荷量（Factor Loadings）

標準化負荷量代表指標與其建構之間的關聯強度。

  • ≥ .70（理想）：λ² = .49，建構解釋了指標約一半的變異
  • ≥ .50（最低門檻）：統計上顯著但仍有較多誤差變異
  • < .50：考慮刪除該題項

工具二：平均變異萃取量（AVE）

AVE = Σλ²i / n = 所有標準化負荷量平方的平均值

門檻：AVE ≥ .50

AVE = .50 表示建構解釋了其指標 50% 的變異量。低於 .50 代表誤差變異大於被解釋的變異。

工具三：組合信度（CR）

CR = (Σλi)² / [(Σλi)² + Σei] ei = 1 - λ²i（各題項的誤差變異）

門檻：CR ≥ .70

CR 與 Cronbach's α 類似但更精確，因為它考慮了不同題項的負荷量差異。

Convergent Validity Assessment in CFA

Confirms that indicators of a specific construct converge, sharing a high proportion of common variance. Three assessment tools:

Tool 1: Factor Loadings

Standardized loadings represent the strength of the relationship between indicator and construct.

  • ≥ .70 (ideal): λ² = .49, construct explains about half the indicator's variance
  • ≥ .50 (minimum): Statistically significant but more error variance than explained
  • < .50: Candidate for deletion

Tool 2: Average Variance Extracted (AVE)

AVE = Σλ²i / n = mean of all squared standardized loadings

Threshold: AVE ≥ .50

AVE = .50 means the construct explains 50% of its indicators' variance. Below .50 means more error than explained variance on average.

Tool 3: Composite Reliability (CR)

CR = (Σλi)² / [(Σλi)² + Σei] ei = 1 - λ²i (error variance per item)

Threshold: CR ≥ .70

CR is similar to Cronbach's α but more accurate since it accounts for different loading magnitudes.

區辨效度（Discriminant Validity）在 CFA 中的評估

確認每個建構都是獨一無二的，能捕捉到其他建構所未能衡量的現象。

方法一：模型比較法（Model Comparison）

將兩個建構之間的相關固定為 1.0（即合併為一個建構），然後比較此一因素模型與原始二因素模型的適配度差異。若適配度顯著不同，支持區辨效度。

限制：此方法不夠嚴格，因為即使高達 .90 的相關仍可能產生顯著的適配度差異。

方法二：Fornell-Larcker 判準（更嚴格的檢驗）

比較兩個建構各自的 AVE 與它們之間相關係數的平方（r²）：

AVEA > r²AB 且 AVEB > r²AB 兩個建構的 AVE 都必須大於它們之間相關的平方

邏輯：潛在建構應該解釋其自身指標的變異量（AVE）要比它與另一個建構所共享的變異量（r²）更多。

交叉負荷的問題

CFA 中若發現顯著的交叉負荷（cross-loadings），即是區辨效度不足的證據。即使允許交叉負荷可改善模型適配度，但這同時承認了建構效度的缺失，應避免這樣做。

Discriminant Validity Assessment in CFA

Confirms that each construct is truly unique and captures phenomena other measures do not.

Method 1: Model Comparison

Fix the correlation between two constructs to 1.0 (merge into one construct) and compare the one-factor model fit with the original two-factor model. Significantly different fit supports discriminant validity.

Limitation: Not strict enough — even correlations as high as .90 can produce significant fit differences.

Method 2: Fornell-Larcker Criterion (More Rigorous)

Compare each construct's AVE with the squared correlation (r²) between constructs:

AVEA > r²AB AND AVEB > r²AB Both constructs' AVEs must exceed the squared inter-construct correlation

Logic: A latent construct should explain more variance in its own indicators (AVE) than it shares with another construct (r²).

Cross-Loading Issues

Significant cross-loadings in CFA indicate discriminant validity problems. Even if freeing cross-loadings improves fit, it simultaneously admits a lack of construct validity and should be avoided.

表面效度與法則效度

表面效度（Face Validity）：在進行任何統計測試之前，研究者必須確認每個題項的內容與建構定義一致。在 CFA 中，這是最重要的效度測試，因為沒有對題項含義的理解，就無法正確指定測量理論。

法則效度（Nomological Validity）：檢驗建構之間的相關是否符合理論預期。CFA 輸出中的建構相關矩陣可用於此評估。

特別注意：使用已發表的量表時，即使這些量表已在其他研究中展現良好的信效度，研究者仍應仔細檢查：當兩個借用的量表放在同一個測量模型中時，是否存在題項內容重疊的問題。

Face Validity & Nomological Validity

Face Validity: Before any statistical testing, researchers must confirm each item's content is consistent with the construct definition. In CFA, this is arguably the most important validity test — without understanding item meaning, one cannot correctly specify a measurement theory.

Nomological Validity: Tests whether correlations among constructs make sense theoretically. The construct correlation matrix from CFA output is useful for this assessment.

Special note: When using previously published scales, even if they showed good reliability and validity in other research, researchers should carefully check for content overlap when two borrowed scales are used together in a single measurement model.

五、模型診斷工具

標準化殘差

觀察共變異與估計共變異的差異除以標準誤。

  • < |2.50|：通常不構成問題
  • |2.50| 到 |4.00|：值得關注但不一定要修改
  • > |4.00|：可能需要刪除有問題的題項
  • 模式比個別值更重要：若某個變數與多個其他變數都有高殘差，這比單一高殘差更值得關注

Differences between observed and estimated covariances divided by the standard error.

  • < |2.50|: Generally not problematic
  • |2.50| to |4.00|: Deserve attention but may not require changes
  • > |4.00|: May need to delete the offending item
  • Patterns matter more than individual values: One variable with high residuals across many others is more concerning than a single high residual

修正指標

顯示釋放（估計）任何未估計路徑後，整體模型 χ² 值的減少量。

  • MI ≥ 4.0 暗示釋放該路徑可顯著改善適配度
  • 但不應僅根據 MI 進行模型修改——必須有理論支持
  • 修正指標是診斷工具，不是模型建構工具
  • CFA 不是探索性工具

Shows how much the overall χ² would decrease by freeing (estimating) any non-estimated path.

  • MI ≥ 4.0 suggests freeing that path would significantly improve fit
  • Model changes based solely on MI are not recommended — must have theoretical support
  • Modification indices are diagnostic tools, not model-building tools
  • CFA is not an exploratory tool

模型修改的注意事項

  • 若超過 20% 的題項被刪除或重新歸類，應使用新的資料集重新驗證
  • 刪除一兩個題項屬於微小修改，通常不會損害確認性測試
  • 即使某題項表現不佳，若刪除後會違反三指標規則或改變量表含義，可保留
  • 理論應始終在模型修改中佔據核心地位
  • If more than 20% of items are deleted or reassigned, a new dataset should be used for re-verification
  • Dropping one or two items is a minor modification that generally does not jeopardize confirmatory testing
  • A poorly performing item may be retained if dropping it violates the three-indicator rule or changes scale meaning
  • Theory should always be prominently considered in model modifications

七、範例演練

範例：員工授權感（Empowerment）四題項模型

四個指標的標準化因素負荷量：

  • X₁：λ = 0.78（這個組織讓我做自己覺得有意義的事情）
  • X₂：λ = 0.89（這個組織讓我有機會在工作中表現卓越）
  • X₃：λ = 0.83（我被鼓勵提出改善組織效能的建議）
  • X₄：λ = 0.87（這個組織鼓勵人們自己解決問題）

Standardized factor loadings for four indicators:

  • X₁: λ = 0.78 (This organization allows me to do things I find personally satisfying)
  • X₂: λ = 0.89 (This organization provides an opportunity for me to excel in my job)
  • X₃: λ = 0.83 (I am encouraged to make suggestions about effectiveness)
  • X₄: λ = 0.87 (This organization encourages people to solve problems by themselves)

計算過程：

λ² 值：0.78² = 0.608, 0.89² = 0.792, 0.83² = 0.689, 0.87² = 0.757

AVE = (0.608 + 0.792 + 0.689 + 0.757) / 4 = 0.712 ✓ (> .50)

Σλ = 0.78 + 0.89 + 0.83 + 0.87 = 3.37

Σe = (1-0.608) + (1-0.792) + (1-0.689) + (1-0.757) = 1.154

CR = 3.37² / (3.37² + 1.154) = 11.357 / 12.511 = 0.908 ✓ (> .70)

結論：聚合效度良好（AVE = .712 > .50, CR = .908 > .70，所有負荷量 > .70）

Calculation:

λ² values: 0.78² = 0.608, 0.89² = 0.792, 0.83² = 0.689, 0.87² = 0.757

AVE = (0.608 + 0.792 + 0.689 + 0.757) / 4 = 0.712 ✓ (> .50)

Σλ = 0.78 + 0.89 + 0.83 + 0.87 = 3.37

Σe = (1-0.608) + (1-0.792) + (1-0.689) + (1-0.757) = 1.154

CR = 3.37² / (3.37² + 1.154) = 11.357 / 12.511 = 0.908 ✓ (> .70)

Conclusion: Good convergent validity (AVE = .712 > .50, CR = .908 > .70, all loadings > .70)
