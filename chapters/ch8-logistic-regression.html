羅吉斯迴歸

Ch.8 當依變數為類別變數時的迴歸替代方案

一、為什麼需要羅吉斯迴歸？

當依變數是二元類別（0/1、是/否、成功/失敗）時，傳統的線性迴歸會遇到三個根本問題：

  • 預測值超出範圍：線性迴歸的預測值可以是任何實數，但機率值必須介於 0 與 1 之間
  • 誤差項不符合常態分配：二元依變數的殘差呈離散分布，違反迴歸假設
  • 異質變異數：殘差的變異數隨預測值變化，違反等變異數假設

When the DV is binary (0/1, yes/no, success/failure), traditional linear regression faces three fundamental problems:

  • Out-of-range predictions: Linear regression can predict any real number, but probabilities must fall between 0 and 1
  • Non-normal residuals: Residuals from a binary DV are discrete, violating regression assumptions
  • Heteroscedasticity: Residual variance changes with predicted values, violating the equal variance assumption

白話解釋

線性迴歸預測「多少」，羅吉斯迴歸預測「是或否」。想像你是一位醫生：線性迴歸可以預測病人的血壓值（連續數字），但如果你想預測病人「會不會得糖尿病」（是/否），就需要羅吉斯迴歸。它不會告訴你一個數字，而是告訴你一個機率——例如「這位病人有 78% 的機率會得糖尿病」。

Linear regression predicts "how much"; logistic regression predicts "yes or no." Imagine you are a doctor: linear regression can predict a patient's blood pressure (a continuous number), but to predict whether a patient "will develop diabetes" (yes/no), you need logistic regression. It provides a probability -- e.g., "this patient has a 78% chance of developing diabetes."

線性迴歸 vs. 羅吉斯迴歸：預測的差異

線性迴歸：Y = b₀ + b₁X → 預測值可能是 -0.3 或 1.5（超出機率範圍）

羅吉斯迴歸：P(Y=1) = ez / (1 + ez) → 預測值永遠在 0 到 1 之間

Linear: Y = b₀ + b₁X → prediction can be -0.3 or 1.5 (outside probability range)

Logistic: P(Y=1) = ez / (1 + ez) → prediction is always between 0 and 1

二、模型原理

羅吉斯迴歸透過logit 轉換，將機率值（0-1 範圍）轉換為不受範圍限制的連續變數，再以線性方程式進行建模。

Logistic regression uses the logit transformation to convert probability (0-1 range) into an unbounded continuous variable, then models it with a linear equation.

核心公式

Logit(P) = ln(P / (1-P)) = b0 + b1X1 + b2X2 + ... + bkXk P = 事件發生機率, P/(1-P) = 勝算 (odds)
P(Y=1) = ez / (1 + ez) = 1 / (1 + e-z) z = b₀ + b₁X₁ + ... ，此為 S 型曲線（sigmoid function）

勝算（Odds）與勝算比（Odds Ratio）：

  • 勝算：P / (1-P)，事件發生與不發生的機率之比。例如：勝算 = 3 表示發生的可能性是不發生的 3 倍
  • 勝算比 Exp(B)：自變數每增加 1 單位時，勝算的倍數變化
  • Exp(B) > 1：自變數增加時，事件發生的勝算增加
  • Exp(B) < 1：自變數增加時，事件發生的勝算減少
  • Exp(B) = 1：自變數對事件發生的勝算無影響

Odds and Odds Ratio:

  • Odds: P / (1-P), ratio of event occurring to not occurring. E.g., odds = 3 means event is 3 times more likely to happen than not
  • Odds Ratio Exp(B): The multiplicative change in odds for each 1-unit increase in the IV
  • Exp(B) > 1: Increasing IV raises the odds of the event
  • Exp(B) < 1: Increasing IV lowers the odds of the event
  • Exp(B) = 1: IV has no effect on the odds

三、模型估計

羅吉斯迴歸使用最大概似估計法（MLE）而非 OLS。MLE 尋找的是讓觀察到的資料最有可能出現的參數值。

Logistic regression uses Maximum Likelihood Estimation (MLE) instead of OLS. MLE finds the parameter values that make the observed data most probable.

-2 Log Likelihood (-2LL)

定義：衡量模型配適度的基本指標。

解釋：數值越小表示模型與資料配適越好。完美配適時 -2LL = 0。

類比：類似線性迴歸中的殘差平方和（RSS）。

-2LL measures model fit. Lower values = better fit. Analogous to RSS in linear regression.

Definition: The fundamental measure of model fit.

Interpretation: Lower values indicate better fit. Perfect fit yields -2LL = 0.

Analogy: Similar to residual sum of squares (RSS) in linear regression.

模型卡方檢定

計算：-2LLnull - (-2LLmodel)

解釋：檢定整體模型是否顯著優於只有截距的零模型。

類比：類似線性迴歸中的整體 F 檢定。

Tests whether the full model is significantly better than the null (intercept-only) model.

Calculation: -2LLnull - (-2LLmodel)

Interpretation: Tests whether the full model is significantly better than the intercept-only (null) model.

Analogy: Similar to the overall F-test in linear regression.

MLE vs. OLS

  • OLS（最小平方法）：最小化殘差平方和。需要連續依變數與常態殘差
  • MLE（最大概似法）：最大化觀察資料出現的概似函數。不需常態假設，適用於類別依變數
  • MLE 採用迭代（iterative）程序，逐步改善參數估計直到收斂
  • 當樣本數夠大時，MLE 的估計值具有漸近常態分配的性質
  • OLS: Minimizes sum of squared residuals. Requires continuous DV and normal residuals
  • MLE: Maximizes the likelihood function of the observed data. No normality assumption, suitable for categorical DVs
  • MLE uses an iterative procedure, improving estimates until convergence
  • With sufficient sample size, MLE estimates are asymptotically normally distributed

四、模型評估

羅吉斯迴歸沒有傳統的 R²，但有多種替代指標來評估模型品質。

Logistic regression lacks a traditional R², but several alternative metrics assess model quality.

評估指標 公式 / 說明 判斷標準
Cox & Snell R² 基於 -2LL 的改善量；上限無法到達 1.0 越高越好（但受限於上限）
Nagelkerke R² 修正 Cox & Snell 使上限可達 1.0 越高越好，較常使用
Hosmer-Lemeshow 將樣本分 10 組，比較預測與觀察的頻率 p > .05 表示模型適配良好
分類表 類似判別分析的混淆矩陣 整體正確率應顯著高於隨機

Pseudo R² 公式

Cox & Snell R² = 1 - [L0 / LM]2/N L₀ = 零模型概似值, LM = 完整模型概似值, N = 樣本數
Nagelkerke R² = Cox & Snell R² / R²max R²max = 1 - L₀2/N，使上限調整為 1.0

注意：Pseudo R² 的限制

Pseudo R² 不能像線性迴歸的 R² 一樣解釋為「解釋變異量百分比」。它們僅是模型改善程度的近似指標，不同的 Pseudo R² 計算方式會得到不同的數值。在羅吉斯迴歸中，分類準確度通常比 Pseudo R² 更具實用價值。

Pseudo R² cannot be interpreted as "percentage of variance explained" like R² in linear regression. They are only approximations of model improvement, and different formulas yield different values. In logistic regression, classification accuracy is usually more practically useful than Pseudo R².

五、係數解釋

羅吉斯迴歸的係數解釋與線性迴歸不同。B 係數代表 logit 的變化，而 Exp(B) 才是最直觀的勝算比解釋。

Coefficient interpretation in logistic regression differs from linear regression. B represents the change in logit, while Exp(B) -- the odds ratio -- is the most intuitive interpretation.

指標 意義 範例
B 自變數每增加 1 單位，logit 的變化量 B = 0.8 → logit 增加 0.8
Exp(B) 勝算比：自變數每增加 1 單位，勝算的倍數 Exp(0.8) = 2.23 → 勝算變為 2.23 倍
Wald 檢定個別係數是否顯著異於零 Wald = (B / SE)²

勝算比 Exp(B) 解讀範例

自變數 B Exp(B) 解釋
年收入（萬元） 0.50 1.65 每增加 1 萬元收入，購買的勝算增為 1.65 倍
年齡 -0.03 0.97 每增加 1 歲，購買勝算減為 0.97 倍（降低 3%）
性別（1=女） 1.10 3.00 女性的購買勝算是男性的 3 倍
品牌忠誠度 0.00 1.00 對購買勝算無影響

Wald 檢定的注意事項

Wald 統計量在 B 係數絕對值很大時會出現膨脹的標準誤，導致本應顯著的係數被錯誤判定為不顯著（Type II error）。當遇到這種情況時，建議改用概似比檢定（Likelihood Ratio Test），比較含與不含該變數的兩個模型的 -2LL 差異。

The Wald statistic can produce inflated standard errors when the absolute value of B is large, causing truly significant coefficients to appear non-significant (Type II error). In such cases, use the Likelihood Ratio Test instead, comparing -2LL values with and without the variable.

六、多元羅吉斯迴歸

當依變數有三個或以上的類別時（如：選擇品牌 A / B / C），使用多元羅吉斯迴歸。其核心概念是選定一個參照類別（reference category），然後針對其他每個類別分別建立一個 logit 方程式。

When the DV has three or more categories (e.g., choose brand A / B / C), use multinomial logistic regression. The core concept is selecting a reference category, then building a separate logit equation for each other category against it.

二元羅吉斯迴歸（Binary Logistic Regression）

依變數只有兩個類別（0 或 1），是最基本且最常用的形式。

  • 僅需建立 1 個 logit 方程式
  • 預測結果：P(Y=1) 的機率
  • 通常以 P = 0.5 作為分類門檻
  • 適用範例：購買/不購買、通過/不通過、有病/無病

Binary Logistic Regression

DV has only two categories (0 or 1), the most basic and common form.

  • Only 1 logit equation needed
  • Predicts: probability of P(Y=1)
  • Typically uses P = 0.5 as the classification cutoff
  • Examples: purchase/no purchase, pass/fail, disease/no disease

多元羅吉斯迴歸（Multinomial Logistic Regression）

依變數有三個或以上的無序類別。

  • 需建立 k-1 個 logit 方程式（k = 類別數）
  • 每個方程式比較一個類別與參照類別的差異
  • 參照類別的選擇不影響模型整體結果，但影響係數的解讀方向
  • IIA 假設（無關替代方案的獨立性）：移除一個選項不應改變其他選項之間的相對機率
  • 適用範例：品牌選擇（A/B/C）、交通方式（公車/捷運/自駕）

Multinomial Logistic Regression

DV has three or more unordered categories.

  • Requires k-1 logit equations (k = number of categories)
  • Each equation compares one category against the reference category
  • Choice of reference does not affect overall model results, but affects coefficient interpretation
  • IIA assumption (Independence of Irrelevant Alternatives): removing one option should not change the relative probabilities among the remaining options
  • Examples: brand choice (A/B/C), transportation mode (bus/metro/car)

IIA 假設：經典的「紅色公車 / 藍色公車」悖論

假設「開車」與「搭公車」各有 50% 的選擇機率。如果新增一條「藍色公車」路線，IIA 假設預測三者的比例為 33% / 33% / 33%。但實際上，藍色公車主要分走的是原公車的乘客，合理的比例應更接近 50% / 25% / 25%。若 IIA 假設被違反，應考慮使用巢狀羅吉斯模型（nested logit）或混合羅吉斯模型（mixed logit）。

Suppose "car" and "bus" each have a 50% choice probability. If a "blue bus" is added, IIA predicts 33%/33%/33%. But realistically, the blue bus mainly draws from existing bus riders, so 50%/25%/25% is more plausible. When IIA is violated, consider nested logit or mixed logit models.

八、關鍵名詞解釋

Logit 轉換（Logit Transformation）

將機率 P 轉換為 ln(P/(1-P)) 的對數勝算。這個轉換將 0-1 範圍的機率值映射到負無窮到正無窮的實數線上，使得可以用線性方程式進行建模。logit 函數是 sigmoid 函數的反函數。

Transforms probability P into the log-odds ln(P/(1-P)). This maps the 0-1 probability range to the entire real number line, enabling linear equation modeling. The logit function is the inverse of the sigmoid function.

勝算比（Odds Ratio）

以 Exp(B) 表示，代表自變數每增加一個單位時，事件發生之勝算的倍數變化。Exp(B) > 1 表示正向影響（增加勝算），Exp(B) < 1 表示負向影響（減少勝算），Exp(B) = 1 表示無影響。勝算比是羅吉斯迴歸中最直觀、最常報告的效果量指標。

Expressed as Exp(B), it represents the multiplicative change in the odds of the event for each one-unit increase in the IV. Exp(B) > 1 = positive effect, Exp(B) < 1 = negative effect, Exp(B) = 1 = no effect. It is the most intuitive and commonly reported effect size in logistic regression.

最大概似估計法（Maximum Likelihood Estimation, MLE）

一種參數估計方法，尋找使觀察到的資料出現機率最大化的參數值。與 OLS 不同，MLE 不最小化殘差，而是最大化概似函數。採用迭代程序，在大樣本下具有良好的統計性質（一致性、漸近常態性、漸近效率性）。

A parameter estimation method that finds values maximizing the probability of observing the actual data. Unlike OLS, MLE maximizes the likelihood function rather than minimizing residuals. Uses an iterative procedure and has desirable large-sample properties (consistency, asymptotic normality, asymptotic efficiency).

Hosmer-Lemeshow 適配度檢定

將預測機率由低到高排列並分成十等分，比較每組的預測事件數與實際觀察事件數之差異。虛無假設為「預測值與觀察值無差異」。p > .05 表示模型適配良好。這是羅吉斯迴歸中少數「希望不顯著」的檢定。

Ranks predicted probabilities and divides them into ten equal groups, comparing predicted versus observed event counts in each. The null hypothesis is "no difference between predicted and observed." p > .05 indicates good fit. This is one of the few tests where non-significance is desired.

IIA 假設（Independence of Irrelevant Alternatives）

多元羅吉斯迴歸的關鍵假設：移除或新增一個替代方案，不會改變其餘方案之間的相對勝算比。違反此假設時（如紅色公車/藍色公車悖論），模型的預測會有偏誤，應考慮使用巢狀羅吉斯（nested logit）或混合羅吉斯（mixed logit）等進階模型。

A key assumption of multinomial logistic regression: adding or removing an alternative does not change the odds ratios among the remaining alternatives. When violated (e.g., red bus/blue bus paradox), predictions are biased. Consider nested logit or mixed logit models as alternatives.
